# 对码说明书 PDF 翻译工作流规则

本项目执行 PDF 翻译时必须以本文件为硬性门禁。规则来自 `japanese-window-covering-localization` skill，并按软件流水线整理。

## 1. 基准

- 以当前英文源 PDF 为唯一基准。
- 不从坏译文继续补丁式修改。
- 每页、每行、每个保护 token 都必须能追溯到源 PDF。
- 输出必须是可编辑 PDF，禁止整页栅格化。
- 用户标注的截图只代表一类问题。修复必须沉淀为脚本级规则、角色规则或 QA 门禁，并扫描当前 PDF 及同模板后续 PDF 的同类页面；禁止只改截图坐标或单个文本框。

## 2. 翻译完整性

- 禁止漏翻、缩减翻译、删减源含义。
- 同一英文原文在不同 PDF 中必须译文一致，除非源文或版式角色不同。
- 加粗英文、引号内按钮/显示值、屏幕值、设备/配件上的英文、`1x`/`2x`/`4x` 不翻译。
- 加粗短字母、短代码、配件数量/规格代码（如 `S`、`S x2`、`16s`、`4x`、`A1`、`OK`、`P1`/`P2`）属于保护 token，不翻译、不改大小、不改粗细，优先保留源文本/源图形。
- 线缆长度等英寸+米格式不得由模型自由翻译，`118"(3m)`、`118ˮ(3m)`、`78.7"(2m)` 等使用固定规则转为 `3m`、`2m`；不得输出缺字方块、叉号或错误引号。
- 相邻但独立的句子不能合并成一个不可单独调整的大块。

## 3. 字体字号

- 禁止任何形式的文字缩放：不得缩小、放大、横向压缩、纵向缩放、按行不同字号或使用 shrink factor。
- 不得全局统一成一个字号。必须按源角色分组：大标题、小标题、正文、警告/编号正文、图例/图片标注、按钮/屏幕值。
- 同级角色字号、字重、行距必须一致。
- 正文与图例/图片标注不是同一角色，不能混用字号。
- 日文字体必须来自配置字体路径，不能临时 fallback 到不受控字体。

## 4. 换行行距

- 译文默认锚定在英文源文本的原始位置和基线。不要先找新位置；先在原位置写入。
- 若同字号译文一行放不下，优先在不遮挡文字、图形、线条和页边安全区的前提下向右延展。
- 向右延展仍放不下时再自然换行，换行必须保持同一左边界、统一行距，并再次检查不得遮挡后续文字或非文字资源。
- 译文横向不能超过当前英文源页的视觉内容安全框（例如标题横线、主体内容栏、用户红框标出的左右边界）。不得用整页宽度或固定页边距作为默认可用宽度；若右侧会越过内容框，必须在框内自然换行。
- 能一行自然放下的不得硬换行。
- 必须换行时，左边界一致、行距均匀、语义断点自然。
- 红色感叹号、数字编号与后续正文距离统一，正文列左对齐，整体垂直居中。
- 最终 PDF 渲染后必须检查新写入译文 bbox：同一段多行不得互相重叠，译文不得压到源页横线、边框、箭头、图形线条或图片。

## 5. 保护符号

- 图标、按钮、屏幕显示值、箭头、三角、太阳/月亮、`CH+`/`CH-`、`P1`/`P2`、`OK`、`A1`/`01`/`02`/`00` 等必须来自当前源 PDF。
- 禁止重画、近似、改粗细、改大小、套统一符号库或从其他 PDF 借图标。
- 符号进入日文括号 token，例如 `「source-icon」`，且与周围文字在同一基线。
- `1x`/`2x`/`4x` 保持源位置、源大小，不裁剪、不重插、不移动。

## 6. 遥控器和图例

- 同一遥控器页面族必须统一译文、字号角色、图标处理和对齐。
- 左侧图例右对齐，右侧图例左对齐，图下说明整体居中。
- 图例、译文和白块不得压到遥控器线条、指示线、屏幕或按钮。
- 图例左右对齐必须按源图形边界和安全间距计算：图片左侧标签以右边缘对齐到图形左侧安全线，图片右侧标签以左边缘对齐到图形右侧安全线；不得直接复用某一页的固定坐标。

## 7. 擦除和非文字资源

- 先擦除源文/坏译文，再写入译文。
- 禁止大白块无脑覆盖；必须尽量 glyph-tight。
- 白块不得遮挡或裁切图片、横线、边框、箭头、遥控器线条、产品图或背景色块。
- 英文残留、旧译文残留、叠字、旧符号残留均为硬失败。

## 8. 可编辑性

- 译文必须是真实可编辑文本。
- 每个独立句子、图例标签、操作短句、状态说明和配件标签默认保持独立可编辑对象。
- 多个邻近文本不得因为距离近而合成一个跨大空白的编辑框。

## 9. QA 门禁

每页必须检查：

- 漏翻
- 叠写/重复翻译
- 同级字号不一致
- 保护符号是否来自源 PDF
- 图片/线条是否被裁切或遮挡
- 行距、左对齐、图例对齐、图下居中
- PDF 文本可编辑

自动检查只能辅助，最终必须基于交付 PDF 的渲染结果逐页审查。

修复后必须重新运行同类问题扫描：如果一个页面出现图例对齐、字号、漏翻、叠写、符号来源、括号内图标、遥控器 P2 页面等问题，必须把同角色/同模板/同页面族全部纳入检查和修复。

## 10. 失败策略

- 同一问题两次局部修复失败，必须从英文源重建该页/块。
- 裁切图片、重叠翻译、漏翻、同级字号不一致、保护符号错误不能放行。
- 无法自动确认的项必须进入报告，不得伪装为通过。
